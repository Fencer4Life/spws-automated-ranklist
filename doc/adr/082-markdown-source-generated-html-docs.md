# ADR-082: Markdown-Source, Generated-HTML Documentation Standard

**Status:** Draft (proposed 2026-07-12; awaiting sign-off)
**Date:** 2026-07-12
**Source:** Documentation tooling — `scripts/render_adrs.py` (ADR archive), `scripts/render_docs.py` (top-level `doc/*.md`), `doc/adr/ADR_TEMPLATE.html`, `doc/adr/assets/adr.css`, `doc/adr/index.html`, `doc/index.html`; CI steps "Validate generated ADR HTML" + "Validate generated Docs HTML".

## Context

The ADR archive was historically **Markdown-only** (`doc/adr/NNN-slug.md`), linked from spec
Appendix C. A separate session introduced an "official Editorial HTML standard": every ADR now also
has a styled `.html` twin generated by `scripts/render_adrs.py`, plus a browsable `index.html`
gallery and shared `assets/adr.css`, each HTML embedding `<meta name="adr-source">` +
`<meta name="adr-source-sha256">`.

This raised an open question — is the `.md` still needed? — touching the standing project preference
that **documents are authored/shipped as styled `.html`, not raw `.md`**. The same question applies
to the rest of the main documentation folder: the Project Specification, the requirements
traceability matrix, the CI/CD operations manual, the development history, and the ingestion design
docs are all read far more comfortably as styled, navigable pages than as raw Markdown. The pipeline
was introduced in code without a decision record. This ADR settles the source/artifact question for
the **whole documentation corpus** and extends the Editorial standard to the top-level docs.

## Decision

Adopt an explicit **source → artifact** model for project documentation — the ADR archive **and**
every top-level `doc/*.md`.

### 1. Markdown is the source of truth

The `.md` is the hand-authored source: edited by hand, git-tracked, and the link target from spec
Appendix C and elsewhere. It is never dropped, moved, or archived — the generators read it in place.

### 2. HTML is a generated artifact

The `.html` is produced **solely** by a generator and never hand-edited. Each embeds provenance — an
`*-source` filename and an `*-source-sha256` of the exact source bytes — so staleness is
machine-detectable.

### 3. Two generators, one shared core

- `scripts/render_adrs.py` renders the ADR archive (`doc/adr/[0-9][0-9][0-9]-*.md` → `.html` +
  `index.html`), and owns the rendering core: markdown-it config, heading IDs + TOC, h2
  section-card layout, link rewriting, HTML validation.
- `scripts/render_docs.py` renders the top-level `doc/*.md` (non-recursive glob — spec, RTM, CI/CD
  manual, development history, ingestion designs, telegram GAS) → sibling `.html`, plus a
  `doc/index.html` hub. It **reuses** the core rather than duplicating or generalizing it, so the
  ADR pipeline stays untouched. Non-recursion excludes `doc/adr/`, `doc/archive/` (superseded),
  `doc/staging/` (generated ingestion logs), and `doc/claude/` (agent-internal).

### 4. Cross-linking

Both generators rewrite local `.md` links to `.html` when the target is in the rendered corpus, so
the styled spec links to the styled RTM and styled ADRs, and the docs browse as a connected set.

### 5. CI enforces sync

`render_adrs.py --check` and `render_docs.py --check` fail the build if any `.html` differs from
what its current `.md` would produce. Both generators are held to ruff + basedpyright.

### 6. Both files committed; edit-then-regenerate workflow

Editing a document: change the `.md` → run the matching generator (ADRs also refresh `index.html`;
docs also refresh `doc/index.html`) → commit the `.md` **and** `.html` together. Furniture
(`ADR_TEMPLATE.html`, the two `index.html`, `assets/adr.css`) is maintained directly.

### 7. Reconciling with "documents are HTML"

The standing preference is really "**readers get styled HTML, not raw Markdown**." The pipeline
honours that — every ADR and every top-level doc ships as styled HTML — while keeping authoring in
Markdown (diff-friendly, greppable, reviewable). Authoring-format and delivery-format are decoupled
by the build step; there is no contradiction. Markdown-source is sanctioned precisely because it
compiles to the required HTML.

### 8. Authoring contract

ADRs: `# ADR-NNN: Title` + an intro block before the first `##` with bold `**Status:**`/`**Date:**`
(optionally `**Source:**`/`**Supersedes:**`/`**Amends:**`). Top-level docs: a leading `# Title`
then `##` sections. No hand-written HTML inside any `.md`.

## Alternatives Considered

- **Markdown-only (status quo ante).** Rejected: no styled/browsable presentation for the ADRs, the
  spec, or the rest of the docs.
- **HTML-only (drop the `.md`).** Rejected: no generator input; forces hand-editing styled HTML;
  breaks `*-source`/sha256 provenance and Appendix C links (which point at `.md`); loses grep/diff.
- **Archive the `.md`.** Rejected: generators read the `.md` in place; relocating the source breaks
  generation. Archiving a live artifact's source is backwards.
- **Generalize `render_adrs.py` into one doc-type-generic renderer.** Rejected for now: the ADR
  pipeline is CI-load-bearing and green; a rewrite risks regressions across 79 files. A sibling
  generator reusing the shared core is lower-risk and equally DRY.
- **Off-the-shelf SSG (MkDocs/Sphinx/Docusaurus).** Rejected as heavier than warranted for ~90
  static docs: adds a toolchain/theme dependency and a build-server model; the bespoke renderers run
  in place with no runtime deps beyond `markdown-it-py`, keeping artifacts co-located and committed.
- **Rendering every `.md` under `doc/` recursively.** Rejected: `doc/staging/` (75 generated
  ingestion logs) and `doc/archive/` (superseded) would produce churn and mislead; the top-level
  glob captures the human-maintained majors.

## Consequences

- Two committed files per document evolving together; a stale HTML is a CI failure, not silent
  drift. This covers the ADR archive and every top-level `doc/*.md`.
- Contributors (human or agent) must never hand-edit a generated `.html`, never delete/relocate a
  source `.md`, and must regenerate before committing a doc change.
- `render_adrs.py` and `render_docs.py` are maintained project infrastructure (lint/type-checked in
  CI); template/CSS changes reflow all documents on the next generate.
- Spec Appendix C keeps linking `.md` (the source); readers wanting the styled view use the `.html`,
  `doc/index.html`, or `adr/index.html`.
- New ADRs are authored as `.md` and rendered; **ADR-082 is itself the first ADR produced under the
  model it documents.**

Enforcement: CI steps "Validate generated ADR HTML" + "Validate generated Docs HTML" (`--check`);
ruff + basedpyright on both generators. No pgTAP/pytest — documentation tooling.
