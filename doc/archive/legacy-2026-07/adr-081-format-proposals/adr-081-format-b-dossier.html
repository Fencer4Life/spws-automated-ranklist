CERT→PROD Event Reconciler

A whole-season diff engine replaces an EVF-only promotion path with verified organizer mapping, explicit field ownership, and guarded deletion.

00 / Failure analysis

Context

The CERT→PROD calendar-promotion path—promote_calendar → fn_import_evf_events—was designed as an EVF special case. That assumption produced three coupled failures:

  1. Organizer hardcoded to SPWS. Seven Samorin rows were promoted with the wrong organizer.
  2. No deletion path. Once CERT removed six duplicates, PROD could not converge without manual intervention.
  3. PEW/MEW-only filter. Incremental promotion excluded domestic PPW/MPW events.

The upstream allocator amplified the problem: blank locations skipped identity reuse and minted a new event code on each scrape.

ARCHITECTURAL AXIOM: season skeleton copy and incremental calendar promotion are one operation at different moments—reconcile PROD’s event set to CERT’s.
01 / Selected design

Decision

Retire fn_import_evf_events. Introduce fn_mirror_events_to_prod(p_creates, p_updates, p_deletes JSONB), a whole-active-season reconciler keyed by txt_code. Python computes the diff; one transactional RPC applies it.

01

Organizer verified from source

CERT organizer codes are resolved to environment-local PROD ids using _prod_code_to_id. The same applies to prior-event references. Unresolved codes raise: fail loudly; never guess.

02

Whole-set diff

No prefix filter remains. By txt_code: CERT∖PROD is CREATE, CERT∩PROD is UPDATE, and PROD∖CERT is DELETE.

03

Field ownership protocol

  • Overwrite: source-owned identity—name, dates, location, country, organizer, weapons, EVF id and slug.
  • Fill blank: admin URLs, fees and registration settings.
  • Never touch: status, tournaments and results.

NULLIF prevents empty CERT strings from entering the unique EVF-slug index.

04

Guarded deletion

Delete only PLANNED events with zero results, dropping empty tournaments first in the same transaction. Results-bearing absences return in delete_skipped and remain intact.

05

Childless creation

CREATE adds no tournament children. The results lifecycle creates tournaments when needed.

06

Active-season equality gate

Abort if CERT and PROD active seasons differ. This prevents new-season creates from being filed into the outgoing PROD season and prevents an apparent deletion of that whole season.

07

Column-agnostic run log

Write doc/staging/reconcile/{season}.{ts}Z.md and upload the same report to CERT storage. Separate applied changes from intentionally unsynced divergences and report convergence.
