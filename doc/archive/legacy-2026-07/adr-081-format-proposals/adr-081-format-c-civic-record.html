CERT→PROD Event Reconciler

An accepted decision establishing one verified, whole-season reconciliation contract between certification and production.

StatusAccepted · shipped
Adopted11 July 2026
EnvironmentsCERT + PROD
Record classData integrity
Statement of circumstances

Context

The calendar-promotion path from promote_calendar to fn_import_evf_events was built as an EVF special case. Its governing assumption produced three related defects:

  1. Hardcoded organizer. Every promoted event received SPWS, producing seven confirmed Samorin mis-tags.
  2. No DELETE. Promotion could create divergence but could not remove it. Six obsolete PROD rows survived after CERT deduplication.
  3. Prefix exclusion. The PEW/MEW filter denied domestic PPW/MPW events an incremental path to production.

A blank-location weakness in fn_allocate_evf_event_code also minted a fresh code on every scrape when location-based reuse was unavailable.

Finding. Season-skeleton copy and incremental calendar promotion are the same responsibility exercised at different times: reconciliation of PROD’s event set to CERT’s.
Resolution

Decision

fn_import_evf_events is replaced by fn_mirror_events_to_prod(p_creates, p_updates, p_deletes JSONB), a whole-active-season Create / Update / Delete reconciler keyed on txt_code. Python owns the diff; SQL applies it transactionally.

Verified organizer identity

The organizer and prior-event codes read from CERT are resolved to PROD-local ids through shared _prod_code_to_id. Missing mappings raise an exception. The system shall fail loudly and shall never guess.

Whole-set comparison

All active-season events participate. CERT∖PROD shall CREATE; CERT∩PROD shall UPDATE; PROD∖CERT shall enter guarded DELETE consideration.

Explicit ownership of fields

  • Source identity is overwritten from CERT.
  • Admin URLs, fees and registration settings are fill-blank-only.
  • Status, tournaments and results remain under the results lifecycle.

Empty strings use NULLIF, preventing invalid empty values in the unique EVF-slug index.

Guarded deletion

An event may be deleted only while PLANNED and free of results. Empty tournaments are removed first, transactionally. Protected absences are reported in delete_skipped.

Childless creation

Created events contain no tournament children. Tournament creation remains demand-driven by results promotion.

Season equality

The operation is refused when active seasons differ. This prevents cross-season misfiling and the appearance that an outgoing PROD season should be deleted.

Public operational record

Each run writes doc/staging/reconcile/{season}.{ts}Z.md and uploads the report to CERT storage. Applied changes and intentionally unsynced divergences are recorded separately with convergence totals.

Effects of adoption

Consequences

  • Self-healing: the first PROD execution removed orphan rows and corrected the surviving organizer.
  • Domestic parity: PPW/MPW events can now be promoted incrementally.
  • Organizer safety: promotion contains no organizer literal.
  • Unified ownership: event C/U/D belongs exclusively to the reconciler.
  • Guarded rollout: active-season mismatch and empty-slug failures have regression coverage.
  • Acceptable cost: season-wide snapshots operate over only tens of rows.
