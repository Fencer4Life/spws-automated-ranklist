02How to run things

Always from the repo root, always with the campaign venv. Paths in the scripts are relative to the repo root and they will exit with a clear message if run from elsewhere.

V=doc/external_files/MSW-Tbilisi-2026/.venv/bin/python
S=doc/external_files/MSW-Tbilisi-2026/scripts

$V $S/roster.py              # headline counts, reads nothing remote
$V $S/auth_setup.py          # re-consent; opens a browser, needs one click
$V $S/send_test.py           # one message to the two operator addresses
$V $S/send_wave1.py          # DRY RUN — writes the preview, sends nothing
$V $S/send_wave1.py --send   # the real thing
$V $S/read_responses.py      # pull declarations from all seven forms
$V $S/gen_raport.py          # rebuild the board report
$V $S/update_variants.py     # push wording changes to all seven forms

03Testing before wave 1 — the plan

The operator asked for a testing pass before the real send. This is the order that finds problems most cheaply. Each step has an expected result; if it does not match, stop and investigate rather than continuing.

TEST 1 · roster sanity roster.py — expect 144 / 173 / 74 / 40 / 103.
If these moved without anyone editing kontakty.md or the workbooks, something upstream changed and nothing else should be trusted until it is understood.
TEST 2 · token is alive and correctly scoped Run read_responses.py. Expect seven lines of 0 response(s).
This exercises auth and all seven form ids in one cheap call. If it complains about scopes, run auth_setup.py. Verify scopes against Google’s tokeninfo endpoint, never against token.json — see the traps section.
TEST 3 · dry run and read the preview send_wave1.py then open wysylka-podglad.html.
Expect 103 to send, 41 skipped. Read the skipped list: every entry should say “brak adresu e-mail”. Any other reason — a missing variant, a category absent from the FIE schedule — is a real bug and must be fixed before sending. Spot-check three or four rows: does the variant match the fencer’s weapons, and does the link open a form showing only those weapons?
TEST 4 · one live message to the operator send_test.py — arrives at both operator addresses.
Check: sender shows as SPWS — Ranking Weteranów; Polish diacritics intact; text reflows on a phone; the cost ladder appears above the weapon questions; the button opens the form with code, name, e-mail and phone prefilled; and it is not in spam. On 19 July it landed in the inbox in both Gmail and Proton.
TEST 5 · submit and read back Fill in the form from that message, then run read_responses.py.
Expect 1 submission -> 1 fencer and matched by kod. If it says nazwisko or e-mail, the token the sender issues does not match the token the reader expects — that is a real defect, and it hid itself once already.
TEST 6 · colours appear gen_raport.py, then open raport.html.
The test fencer should be green (or hatched green on the team row). Everyone else stays white. The banner should still say nothing was sent, because no send_log.jsonl exists yet.
TEST 7 · a real but tiny send send_wave1.py --send --limit 1, having temporarily put an operator address at the top of the queue, or simply accept that one real fencer is mailed early.
Confirms the log is written, the pacing works, and a resume skips the person already sent. Re-run the same command — it must report already sent 1 and mail nobody twice.
Clean up after testing Delete any test submissions from each form’s Responses tab (⋮ → Delete all responses) before the real wave, and delete send_log.jsonl if the test send should not count. After the real send, never use “Delete all responses” again — it destroys genuine declarations with no undo.

05Traps — each of these cost real time

A scope check that validated itself. Credentials.from_authorized_user_file(path, SCOPES) sets .scopes from the argument, not the file. Comparing that to SCOPES always passes, and to_json() then writes the false claim into the token file. Always verify scopes against https://oauth2.googleapis.com/tokeninfo — an external authority with no stake in our assumptions. gauth.stored_scopes() now reads the JSON directly.
A token mismatch hidden by a fallback. The test script hard-coded a name spelling that differed from the roster’s, so it generated a different hash. The join still worked — via name matching — and looked healthy. This is why read_responses.py records matched_by. A fallback that does not say it fired is a defect generator.
A superseded form left live. The original all-weapons form stayed fillable after the seven variants replaced it, and an old e-mail pointed at it. Nothing reads that form, so a submission there would have vanished silently and the fencer would have been chased for not replying. It is now emptied and retitled NIEAKTUALNY FORMULARZ. Superseding is finished only when the old thing cannot be used by accident.
Identical test subjects. Six test messages with the same subject line meant the wrong one got opened, pointing at the wrong form. send_test.py now stamps the subject with the time and the variant key.
A per-team fee written as per-fencer. The ladder said “każda drużyna — dodatkowe 185 EUR”, which reads as a personal charge and overstated an individual’s cost roughly fourfold. Corrected everywhere. Compressing a quoted figure into computed arithmetic changes what it asserts.
A constraint inherited without checking its scope. “Forms cannot vary questions per respondent” is true — but it does not follow that everyone gets the same form. That unexamined leap produced a form showing three weapons to a one-weapon fencer. Check what a limitation actually forbids.
Diacritics in name matching. NFKD plus ASCII-encode deletes ł entirely, so “Paweł” becomes “pawe”. Polish letters must be transliterated first, via str.maketrans. Preserved in roster.key() — do not simplify it.
Duplicate names collapsing to the wrong row. setdefault keeps the first entry, which was the empty one for two fencers, reporting them unreachable. roster._contacts() prefers the populated row. When collapsing duplicates, prefer data over position.

06Safety rails already in the code

These exist so a mistake is loud rather than silent. Do not remove them to make something more convenient.

  • Dry run is the default. send_wave1.py mails nobody without --send.
  • The log is written before the next send. A crash mid-run loses no record; re-running skips everyone already sent.
  • One message per recipient. No fencer ever sees another’s address. Even the two-address test rehearses this shape.
  • Deterministic tokens. A resend reissues the same code, so a retry never creates a second identity for one person.
  • Wrong-account refusal. auth_setup.py exits rather than writing a token for any account other than the campaign one.
  • Unknown schedule entries raise. A fencer in a category Tbilisi does not run fails loudly instead of being mailed an invented date.
  • build_variants.py refuses to re-run. A second set of forms would split responses irrecoverably.
  • Answers beat inferences. In gen_raport.state(), a fencer who replied is never amber and an unreachable fencer is never amber — so amber always means real work.

08Working style that the operator expects

  • English for everything written to him — plans, analysis, chat. Polish only for what reaches fencers: e-mails, forms, the board report.
  • HTML for deliverables. Chat stays short: a pointer and one question, never long prose or inline tables.
  • Act, do not ask twice. Asking a question already answered, or asking permission for work that is plainly in scope, is worse than a wrong default that is stated plainly and easy to reverse.
  • Verify, then claim. Read the state back from the source — Google, the file, the script — rather than reporting what the code intended.
  • Name uncertainty. He reads carefully and catches things. An unflagged guess costs far more trust than an admitted unknown.
