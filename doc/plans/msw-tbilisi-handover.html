01What this is

A one-off information campaign to find out which Polish veteran fencers intend to compete at the FIE Veteran World Championships in Tbilisi, 9–17 October 2026. It is deliberately outside the ranking system: no ADR, no migrations, no handbook page, no repo integration. It exists to get a job done faster with automation.

Final nomination is made by PZSz, not SPWS — SPWS only proposes. That framing is in the email and matters.

05Mistakes I made — do not repeat, do not trust

Listed because the next session will otherwise inherit the same blind spots, and because the user is right to be sceptical of anything here he has not checked.

1. Invented a form capability that does not exist. I designed and drew four detailed form mock-ups in which each question shows the fencer's own ranking position, start date and cost. Google Forms cannot vary question text per respondent — only prefilled answers. Caught only when I sat down to write the build spec. The affected document is msw-tbilisi-formularz-przyklady.html; its mock-ups are wrong. Lesson: check what the tool can do before designing around it.
2. Name matching silently dropped Polish characters. unicodedata.normalize("NFKD") plus ASCII-encode deletes ł entirely, so "Paweł" became "pawe" while "Pawel" became "pawel" — different keys. Harmless while both sources spelled names identically, then broke the moment the second contact list mixed spellings. All scripts now translate Polish letters first via str.maketrans. Lesson: normalisation that happens to work is not normalisation that works.
3. Duplicate-name merge took the wrong row. contacts.setdefault(key, …) kept the first entry for a name, and for both "Krawczyk Paweł" and "Młynek Janusz" the first entry was the empty one — so two reachable fencers were reported as unreachable. Unreachable count was wrong at 43 for several turns; it is 41. Fixed by preferring the populated row. Lesson: when collapsing duplicates, prefer data over position.
4. Carried a fee the user had already cancelled. He told me Polish citizens need no visa and there is no 55 EUR fee. It nonetheless survived in several documents and reappeared in a draft. Purged now — the phrase appears nowhere. Lesson: when a fact is retracted, grep the whole corpus, not the document in front of you.
5. Nearly committed personal data to the repository. The four form mock-ups in msw-tbilisi-formularz-przyklady.html used real fencers with their real email addresses and phone numbers. That file is tracked. Caught by a scan of the staged diff seconds before the commit, not by design. Contact fields there now read —— e-mail ——. Lesson: scan every tracked document for addresses and phone numbers before staging — the gitignore protects the data directory, not documents that quote from it.
6. Repeatedly ignored a stated format rule. The user told me more than once that he reads HTML and that questions belong in the document, not the chat. I kept producing long chat prose and inline tables anyway. This is now recorded in memory as feedback_all_output_html. Lesson: a stated preference is a constraint, not a suggestion.
One thing I got right that is worth keeping Every count in these documents is produced by a script over the source workbooks, not typed by hand. That is why errors 2 and 3 were findable at all. Keep that property: if a number appears in a document, a script must have produced it.
