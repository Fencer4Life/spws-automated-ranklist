Frontend

Pipeline and tooling

Parses sources, resolves identity, orchestrates validation/commit, builds reports, synchronizes EVF, recomputes affected events and promotes approved data.

Supabase

Domain and security boundary

PostgreSQL owns entities, invariants, transactional writes, rankings and public read models; Auth protects administration; Storage holds selected artifacts; Edge Functions dispatch trusted actions.

GitHub Actions

Execution and delivery

Runs CI, release, ingestion, synchronization, promotion, report generation and queue draining with environment-scoped secrets.

GAS / Telegram

Operational adapter

Receives email artifacts or commands, uploads/dispatches work and returns human feedback without becoming the domain database.

SPWS host

Embedding container

Hosts association content and embeds isolated public custom elements produced by the frontend build.

Access contract

Reads and writes

Reads

Public and frontend reads use curated views and RPC results such as calendar, ranking, score and entry-list projections. These are presentation contracts, not permissions to infer raw-table shape.

Writes

Business writes use SECURITY DEFINER functions, ingestion commit RPCs, authenticated admin RPCs or trusted workflows. Anonymous direct writes to core tables are not the application contract.

Current composition

Intentional transition seams

The declarative plugin engine is active, but some plugins delegate to existing stage functions through a bridge. This is not two independent pipelines: the bridge is a compatibility seam allowing the new orchestration contract to reuse proven domain logic while migration continues.

Draft review and live automated flow are related but distinct: operator-reviewed drafts use run-scoped draft tables and a commit RPC, while the plugin flow provides reusable automated sequencing and post-commit reactions.

Architecture rules

Cross-container invariants

Database ids are environment-local; promotion resolves stable codes rather than copying raw ids.
Parsers emit source facts; canonical event metadata and identity decisions are applied later.
Frontend display logic cannot override database scoring or authorization.
Domain faults are distinguishable from infrastructure aborts.
CERT and PROD changes remain explicit and auditable.
