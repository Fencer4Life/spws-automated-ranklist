Security model

Trust boundaries

Public boundary

  • Anonymous reads use curated views/RPCs.
  • Registration accepts constrained public input.
  • Embedded components receive no deployment credentials.

Trusted boundary

  • Admin actions require authenticated MFA sessions.
  • Workflow dispatch credentials live behind Edge Functions/GitHub secrets.
  • CERT/PROD mutations occur through explicit workflows and promotion contracts.
Credential rule: PATs, service-role keys and deployment secrets never belong in frontend bundles, repository files or workspace MCP configuration.

Deployment topology

Environment contract

LOCAL is the development and operator workspace. CERT is the certification surface for review and parity. PROD is the published production system. Schema and software move through controlled release; selected event/season/result data moves through explicit promotion or reconciliation.
