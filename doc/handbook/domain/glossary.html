Competition model

Core entities

Season
A bounded ranking period with dates, active-state selection and one scoring configuration.
Event
A calendar-level competition occurrence. It owns identity, dates, location, organizer, lifecycle and links to prior-season counterparts.
Tournament slot
The scoring destination beneath an event: one weapon, gender and veteran age category, with participant count and result source.
Result
A fencer’s placement in a tournament slot, linked by foreign key to the canonical fencer and carrying source/match provenance.
Registration
A person’s entry request for an event, including identity-matching state and consent evidence. It is not a competition result.
Scoring configuration
Season-specific parameters controlling points, eligible counts, international inclusion, participant thresholds and carry-over behavior.

Event families

Common codes

SPWS
Stowarzyszenie Polskich Weteranów Szermierki, the Polish Veterans Fencing Association and domestic data owner.
PPW
Domestic individual ranking event family.
MPW
Domestic championship event family.
PEW / MEW / MSW
International/European or championship event families used in the extended ranking pool. Exact eligibility is governed by scoring configuration and ADRs.
PSW
SPWS-organized school/event family that follows domestic participant-threshold policy.
DMEW / IMEW
Team/biennial event concepts with special calendar or carry-over behavior; not every displayed event contributes identically to rankings.

Ranking dimensions

Weapons, genders and categories

Weapon
EPEE, FOIL or SABRE.
Gender
M or F in the ranking model. Mixed fencing may occur physically, but ranking destinations remain gender-specific.
V0
Age 30–39 in the season end year; a domestic SPWS category without an EVF/FIE veteran counterpart.
V1
Age 40–49 in the season end year.
V2
Age 50–59 in the season end year.
V3
Age 60–69 in the season end year.
V4
Age 70 or older in the season end year.

Data processing

Pipeline terms

Source
One of the supported parser kinds: FencingTime XML, FTL, Engarde, 4Fence, Dartagnan, EVF API, file import, Ophardt HTML or CERT reference data.
IR / ParsedTournament
The source-neutral Python contract produced by parsers before event metadata, identity and database decisions are applied.
Draft
Reviewable tournament/result state associated with a run id before transactional commit to live tables.
Plugin
A declared pipeline unit with reads, writes, effects and a kind such as source, gate, transform, mutator or reactor.
Flow / Rule
An ordered, validated plugin plan for one scenario such as domestic ingest, recompute, dedup sweep or post-commit handling.
Fault
A recoverable domain problem recorded in pipeline context; it does not automatically mean infrastructure failure.
Abort
A genuine infrastructure/programming failure that stops the flow and is eligible for retry or repair.
Parity
A comparison between independently sourced or computed data used to decide whether promotion or publication is trustworthy.

Delivery

Environment terms

LOCAL
Developer/operator environment used for schema, pipeline and UI work. Database reset must use the repository reset script.
CERT
Certification environment where data and releases are reviewed before production promotion.
PROD
Production environment embedded by the SPWS site and protected from direct experimental mutation.
Promotion
An explicit movement or reconciliation of approved schema, events, seasons or results from CERT toward PROD.
