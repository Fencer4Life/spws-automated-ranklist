How it works: (1) Admin clicks "Import EVF" → fetches veteransfencing.eu/calendar. (2) Events within selected season's date range are shown. (3) Already-imported events are grayed out with "ZAIMPORTOWANE" tag. (4) Admin checks new events to import → "Importuj N wydarzenia". (5) Each imported event creates a tbl_event row with PLANNED status + child tbl_tournament entries based on weapons.

Field Mapping

EVF Calendar Field → tbl_event Column Notes
Event name → txt_event_name As-is from EVF
Start date → dt_event_date First day of event
City → txt_city Extracted from location
Country → txt_country (M8 new) From venue address
Venue + address → txt_venue_address (M8 new) Full address string
Registration link → url_invitation (M8 new) EVF event page URL
Price → num_entry_fee (M8 new) Numeric value (€)
Weapons list → tbl_tournament children Create PEW/MEW tournaments per weapon
— → enum_event_status Always PLANNED on import
— → enum_tournament_type Always PEW/MEW (international)
Deduplication: Events already in tbl_event for the target season are matched by name similarity (fuzzy) + date proximity (±7 days). Matches are shown as "ZAIMPORTOWANE" and cannot be re-imported. This prevents duplicate entries.
EVF import is an admin-only feature (M8 scope). Tournament child creation is automatic based on weapons detected.
