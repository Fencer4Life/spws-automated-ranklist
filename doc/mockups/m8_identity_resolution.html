Design Notes

  • Main queue shows all tbl_match_candidate rows, filterable by status and tournament.
  • Default filter: PENDING only — the items requiring admin action.
  • Stats bar shows breakdown across all statuses for the current season.
  • Confidence % from num_confidence (0–100). Color-coded: green ≥95, yellow 50–94, red <50.
  • Three admin actions:
  • Zatwierdź / Wybierz → opens Approve modal. Shows fencer candidates with radio buttons. "Wybierz" label when multiple candidates need disambiguation.
  • Nowy zawodnik → opens Create New Fencer form. Pre-filled from scraped name. Birth year estimated (yellow warning). Only for domestic (PPW/MPW) tournaments.
  • Odrzuć → opens Dismiss confirmation with reason dropdown and optional note. Used for international fencers not in SPWS, duplicates, bad data.
  • Disambiguation (Part 3): when 2+ fencers share the same name, the modal highlights which one matches the age category (V0–V4 bracket shown).
  • AUTO_MATCHED rows are dimmed (opacity) in the queue — resolved, shown for reference only.
  • UNMATCHED + international (PEW/MEW/MSW): only Dismiss available (cannot create fencer for international events).
  • UNMATCHED + domestic (PPW/MPW): "Nowy zawodnik" button available (auto-create from intake rules).
  • Admin note: optional free-text stored in txt_admin_note for audit trail.
  • Aliases shown in candidate list — helps admin recognize alternative spellings (e.g., "TK" alias for KOŃCZYŁO Tomasz).
