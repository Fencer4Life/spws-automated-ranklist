Design Notes

  • Two import paths:
  • Event-level ⬇ Import (on event header) — opens modal with tournament checklist, uses the event URL, imports multiple tournaments at once.
  • Tournament-level ⬇ (on tournament row) — opens modal for that single tournament, uses the tournament's own URL or file upload.
  • Both modals have two tabs: 🔗 Z adresu URL (scrape) and 📁 Z pliku (upload Excel/JSON/CSV).
  • Single tournament modal shows an editable URL field (the tournament's own results URL, separate from the event URL).
  • Status tags:
  • NOWY (green) — first import, no existing results
  • REIMPORT (yellow) — has existing results, will delete + re-import in transaction (ADR-014), warning shown
  • Tournament row actions: ⬇ Import (single), ✏️ Edit (name/code/URL/type), 🗑 Delete (cascade results)
  • + Dodaj turniej inside expanded event — manual tournament creation (admin defines weapon, category, URL, etc.).
  • Event-level ⬇ Import for batch import from event URL (multi-select checklist).
  • Event-level ✏️ edits event fields (name, date, city, status, + 4 new M8 columns).
  • Event-level 🗑 deletes event + all child tournaments and results (cascade).
  • Footer summary shows count + breakdown (how many reimports vs new) in event-level modal.
