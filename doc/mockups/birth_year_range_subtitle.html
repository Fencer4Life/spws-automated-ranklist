Implementation notes:
• No DB change needed — computed from season.dt_end year and the hardcoded age thresholds (30/40/50/60/70) already in fn_age_category.
• Formula: newest = end_year - min_age, oldest = end_year - (min_age + 9). V4 is open-ended: "i starsi".
• Display format: newest, newest-1, .. oldest — enumeration style with ellipsis.
• Reactive: updates when season or category selector changes. Compare Variant A (2025-2026, V2: 1976..1967) vs Variant D (2024-2025, V2: 1975..1966).
• i18n: birth_year_enum key with {cat}, {newest}, {oldest} placeholders. V4 uses birth_year_enum_oldest with "i starsi" suffix.
