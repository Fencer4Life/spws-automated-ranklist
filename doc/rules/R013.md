# R013: Min-participant ingestion gate

## Statement
Tournaments with fewer competitors than the season's configured
threshold are skipped at ingestion. The threshold lives in
`tbl_scoring_config.int_min_participants_ppw` (PPW / MPW / PSW) and
`tbl_scoring_config.int_min_participants_evf` (PEW / MEW / MSW). Strict
less-than semantics: `n_competitors < threshold` ⇒ skip. The default
PPW threshold is `1` (every bracket, including 1-competitor walkovers,
counts); the default EVF threshold is `5`.

## Why
SPWS rules award ranklist points to the lone competitor in a 1-person
bracket (walkover). Before ADR-066 the inclusion logic was hardcoded
to `n_results == 0` in two runners, which meant FTL's null-place
1-competitor JSON was silently dropped — so the lone competitor got
no points despite the rules. Wiring the threshold to the existing
`int_min_participants_*` config knob makes the inclusion policy
configurable per season without code edits, and gives operators a
single place to raise the bar if SPWS rules change.

## Trigger
- Phase 5 runner (`python/tools/phase5_runner.py`): per-bracket
  gate after parsing, before draft write.
- Active-season orchestrator (`python/tools/recreate_active_season_2025_2026.py`):
  same gate, same helper.

## Edge cases
- **Walkover precedence vs `excluded=True`**: an entry with
  `excluded=True` is dropped before the walkover rule checks `n=1`. A
  bracket with one entry, that entry excluded, yields zero results
  (walkover does not activate).
- **Multi-entry-with-all-null-places**: stays as today — drop all
  entries. Walkover is strictly for `n_non_excluded == 1` brackets.
- **Threshold raised mid-season**: previously-ingested tournaments
  stay in the DB (the gate runs at ingestion only). Operators must
  manually delete pre-existing below-threshold tournaments if they
  want them removed from the ranklist after a threshold raise.
- **Unknown tournament type / orphan season**: gate defaults to
  threshold=1 (include everything). Conservative — better to ingest
  a borderline bracket than silently drop it.
- **PSW classification**: PSW (Mistrzostwa Szkolne — school events,
  SPWS-organized domestic) uses `int_min_participants_ppw`. Stand-alone
  PSW threshold column is **not** introduced.
- **DMEW (team)**: routed to MPW threshold for ingestion purposes,
  but DMEW results don't enter the rolling-score ranklist (per
  `project_dmew_imew_alternation.md`); the gate matters only for
  pipeline consistency.
