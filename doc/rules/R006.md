# R006: Identity match — auto-create domestic, skip international

## Statement
Identity resolution outcomes depend on tournament organizer:

- **Domestic (SPWS — PPW/MPW):** confidence ≥95 → AUTO_MATCH; 50–94 →
  surface in event diff for admin pick (USER_CONFIRMED on approval);
  <50 → SPWS auto-creates a fencer with estimated birth year
  (AUTO_CREATED + `bool_birth_year_estimated = TRUE`). All rows land
  in `tbl_result`.
- **International (EVF/FIE — PEW/MEW/MSW/PSW, POL-only per R005):**
  confidence ≥95 → AUTO_MATCH; 50–94 → surface in event diff; <50 →
  **skip the row entirely** — no fencer auto-creation, no `tbl_result`
  write.

`enum_match_method` records the provenance per `tbl_result` row.

## Why
SPWS owns the domestic master fencer table and is committed to
"every domestic result counts" — even if a fencer's identity isn't
confidently resolvable, they get a placeholder identity so their
score appears. International events are different: the ranking pool
is POL-only, fencer count is large, and auto-creating from a
low-confidence international match would bloat the master with names
that may never reappear. Skipping is the correct cost trade-off.

## Edge cases
- A scraped name that hits an alias (`tbl_fencer.json_name_aliases`)
  goes straight to AUTO_MATCH at confidence 100, regardless of fuzzy
  score against the canonical name.
- Disambiguation tie-break (multiple same-name fencers): falls back
  to V-cat fit; if no fit, surfaces as USER_CONFIRMED candidate.
- Auto-created fencer (AUTO_CREATED) reuses any subsequent matches
  to the same scraped name via alias write-back.
