# R005b: EVF/FIE V0 prohibition

## Statement
V0 (30–39 age range) is a domestic SPWS-only category — EVF and FIE
events do not include V0 (their veteran categories start at 40+).
The pipeline MUST halt with a clear error if it encounters a V0 result
in an EVF- or FIE-organized event. This is a data-corruption signal,
not a routine condition to swallow.

## Why
Per ADR-008, V0 has no international counterpart. If a parser produces
a V0 row from an EVF source, something is wrong — either the source
mis-tagged the category, the V-cat splitter (R001) miscategorized a
fencer, or the event is mis-classified as EVF. Halting forces the
admin to investigate; silent drop would mask the upstream bug.

## Trigger
Stage 6 (identity resolution + V0/EVF check) of the unified pipeline:
fires when `event.id_organizer ∈ {EVF, FIE}` AND any
`ParsedResult.source_vcat_hint == 'V0'` (or splitter assigns V0 to a
result).

## Edge cases
- A POL fencer young enough to be V0 fencing in EVF: per ADR-008 they
  cannot be in an EVF veteran event; their participation is invalid
  upstream. Pipeline still halts.
- Combined-pool source where V0 is implicit (e.g. v0v1 marker on EVF):
  R001 splitter would produce a V0 slice — Stage 6 then halts. This is
  the intended behavior — the EVF event was mis-tagged with v0v1.
