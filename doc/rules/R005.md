# R005: EVF POL-only filter

## Statement
For events organized by EVF (`tbl_event.id_organizer = EVF`), the
ingestion pipeline imports results for **Polish (POL) fencers only** —
non-POL rows are skipped at parse time, never landing in `tbl_result`.
The international ranking pool (Kadra) is built from POL participation
in EVF events; foreign fencers' placements are not Polish ranking data.

## Why
EVF-organized events can have hundreds of fencers from multiple
countries. Importing all of them would (a) pollute identity resolution
with names that will never match SPWS master data, (b) bloat ranking
views with rows that get filtered out anyway, (c) make the matcher's
"low confidence" zone meaningless. POL-only intake is the correct
upstream filter.

## Trigger
Stage 6 (identity resolution) of the unified pipeline:
- For EVF-organized events: rows with `parsed_country != 'POL'` are
  dropped before fuzzy matching.
- For SPWS-organized events: filter does not fire.

## Edge cases
- Source missing country information (e.g. older Engarde formats):
  the pipeline must FAIL the event rather than silently importing
  everyone or no-one. Admin override required.
- Polish fencer abroad on a foreign club registered as `FRA`: still
  POL by nationality. Country detection uses `parsed_country` from
  the IR (parser's job to extract national affiliation, not club).
- See R005b for V0 + EVF combination — separate prohibition.
